# Metameros Network Repository Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Detach the current public GitHub fork, preserve its Git and GitHub evidence, and rename the resulting standalone repository to `VellichorVoyager/metameros-network`.

**Architecture:** Treat the migration as a guarded transaction with two verified external backups before the irreversible GitHub operation. Preserve Git authorship and upstream provenance, then restore only supported repository metadata after GitHub reports the repository as standalone.

**Tech Stack:** Git, GitHub CLI, GitHub REST API, GitHub web settings, Bash, SHA-256, Markdown

---

- [ ] **Step 3: Export repository-level GitHub metadata**

Run:

```bash
REPO="VellichorVoyager/TMO-G5AR-Portal"
gh api "repos/$REPO" > "$BACKUP_DIR/metadata/repository.json"
gh api "repos/$REPO/topics" > "$BACKUP_DIR/metadata/topics.json"
gh api --paginate --slurp "repos/$REPO/labels?per_page=100" | jq 'add' > "$BACKUP_DIR/metadata/labels.json"
gh api --paginate --slurp "repos/$REPO/milestones?state=all&per_page=100" | jq 'add' > "$BACKUP_DIR/metadata/milestones.json"
gh api "repos/$REPO/actions/permissions" > "$BACKUP_DIR/metadata/actions-permissions.json"
gh api "repos/$REPO/actions/variables" > "$BACKUP_DIR/metadata/actions-variables.json"
gh secret list --repo "$REPO" --json name,updatedAt > "$BACKUP_DIR/metadata/actions-secret-names.json"
gh api "repos/$REPO/environments" > "$BACKUP_DIR/metadata/environments.json"
gh api "repos/$REPO/community/code_of_conduct" > "$BACKUP_DIR/metadata/code-of-conduct.json" 2> "$BACKUP_DIR/metadata/code-of-conduct.error" || true
```

Expected: required API captures are valid JSON; unavailable optional data has an adjacent error record.

- [ ] **Step 4: Export all pull requests, discussions, and patches**

Run:

```bash
gh api --paginate --slurp "repos/$REPO/pulls?state=all&per_page=100" | jq 'add' > "$BACKUP_DIR/metadata/pulls.json"
jq -r '.[].number' "$BACKUP_DIR/metadata/pulls.json" | while read -r number; do
  mkdir -p "$BACKUP_DIR/metadata/prs/$number"
  gh api "repos/$REPO/pulls/$number" > "$BACKUP_DIR/metadata/prs/$number/pull.json"
  gh api --paginate --slurp "repos/$REPO/pulls/$number/reviews?per_page=100" | jq 'add' > "$BACKUP_DIR/metadata/prs/$number/reviews.json"
  gh api --paginate --slurp "repos/$REPO/pulls/$number/comments?per_page=100" | jq 'add' > "$BACKUP_DIR/metadata/prs/$number/review-comments.json"
  gh api --paginate --slurp "repos/$REPO/issues/$number/comments?per_page=100" | jq 'add' > "$BACKUP_DIR/metadata/prs/$number/issue-comments.json"
  gh api -H 'Accept: application/vnd.github.patch' "repos/$REPO/pulls/$number" > "$BACKUP_DIR/metadata/prs/$number/pull.patch"
done
```

Expected: 37 numbered pull-request directories and no failed required API calls.

- [ ] **Step 5: Export releases and wiki data**

Run:

```bash

gh release view v0.2.0 --repo "$REPO" --json tagName,name,body,isDraft,isPrerelease,createdAt,publishedAt,url,assets > "$BACKUP_DIR/releases/v0.2.0.json"
jq -r '.body' "$BACKUP_DIR/releases/v0.2.0.json" > "$BACKUP_DIR/releases/v0.2.0-notes.md"
mkdir -p "$BACKUP_DIR/releases/v0.2.0-assets"
```

Expected: release JSON and notes exist; an absent wiki or absent release assets are explicitly recorded and do not invalidate the Git mirror.

- [ ] **Step 6: Scan archive filenames and content for common secret patterns**

```bash
  '(gh[pousr]_[A-Za-z0-9_]{20,}|tskey-[A-Za-z0-9-]+|SHODAN_API_KEY=.+|BEGIN (RSA |EC |OPENSSH )?PRIVATE KEY)' \
```

- [ ] **Step 7: Generate and verify checksums**

Run:

```bash

  | sort -z \
  | xargs -0 shasum -a 256 \
  | sed "s#  $BACKUP_DIR/#  #" > "$BACKUP_DIR/SHA256SUMS"
(cd "$BACKUP_DIR" && shasum -a 256 -c SHA256SUMS)
```

Expected: every listed artifact reports `OK`.
